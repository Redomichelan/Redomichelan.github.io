算法分析与设计——集合(5)众数问题pro

2021-03-26- 2021-04-05
算法课程-算法分析与设计课程集合
Redomichelan

数组中没有负数，求数组的众数，如果存在多个数组或者所有元素都不同则输出最左边的数字，要求空间复杂度O(1)

问题描述

给定含有n个元素的数组S,数组元素个数在50万以内。每个元素在S中出现的次数称为该元素的重数。数组S中重数最大的元素称为众数。 求下列情况的众数。注意时间和空间复杂度要求。

数组中没有负数，求数组的众数，如果数组中存在重数相同的多个数字或者所有元素都不同，则按照数字出现的顺序，只输出最左边的数字。 要求空间复杂度为O(1)。

输入格式

输入一个数组，以逗号为间隔

输入样例

10,34,23,23,34,34,23,34,23

输出格式

输出该数组中众数

输出样例

34

解题思路

这题我想了很久，因为要求空间复杂度为O(1)，但是数组建立就是需要空间O(n)的，因为这题和leecode不一样，leecode会直接传参一个数组，但是这里需要自己编写main函数。自己编写的情况下不用数组挺难的，当然也可能是我想多了，可能输入成数组不需要考虑。然后我用Stringbuilder解了，空间复杂度应该不高，但是时间复杂度不低。大概思路是，将输入的字符串存入Stringbuilder，在字符串最后加一个’,’，然后进行遍历，每次以逗号为分界读取一个数字加逗号的字符串，再删除所有相同的字符串，通过计算的到该序列的重复次数，这样依次进行删除，直到字符串长度为0。这题应该有其他解法，但是我想不出来。

解题步骤

  1. 将字符串存入StringBuilder，再末尾添加一个’,’。
  2. 建立函数getstr，功能能够获取当前字符串的首个数字序列，以逗号为界，比如“123,”
  3. 建立函数remove，功能是删除目标字符串中的所有子序列，并通过前后字符串之差计算并返回删除序列的数目。
  4. 在遍历中得到最长的序列并删除逗号进行输出。

解题代码

import java.util.Scanner;
public class Step5 {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);
        String strOri = scan.nextLine();
        StringBuilder stropra = new StringBuilder();
        stropra.append(strOri);
        stropra.append(',');
        int maxnum = 0;
        StringBuilder maxstr = new StringBuilder();
        StringBuilder strTemp = new StringBuilder();
        while ( stropra.length() != 0){
            getStr(stropra,strTemp);
            int countTemp = countNum(stropra,strTemp);
            if (maxnum<countTemp){
                maxnum = countTemp;
                maxstr.setLength(0);
                maxstr.append(strTemp); }
            strTemp.setLength(0); }
        System.out.print(maxstr.substring(0,maxstr.length()-1));
    }
    public static int countNum(StringBuilder strTar,StringBuilder substr){
        int count = 0;
        int flag = 0;
        while (strTar.indexOf(String.valueOf(substr), flag-1)!=-1){
            flag = strTar.indexOf(String.valueOf(substr));
            strTar.delete(flag+1,flag+substr.length());
            count++;
        }
        strTar.delete(0,substr.length()-1);
        substr.delete(0,1);
        count++;
        return count;
    }
    public static void getStr(StringBuilder strTar,StringBuilder sub){
        int i =0;
        sub.append(",");
        while(i<=strTar.indexOf(",")){
            sub.append(strTar.charAt(i));
            i++; } }
}

个人评价

空间复杂度应该是满足了，但是代码质量很差，完全没有体现算法的意义。我知道用正则应该更快一些，但是我根本就不想去优化它，只能等之后再用别的方法写了。
