phase_5

解题过程

查看阶段五伪代码

unsigned int __cdecl phase_5(int a1)
{
  int v1; // eax
  char v3[6]; // [esp+15h] [ebp-17h]
  char v4; // [esp+1Bh] [ebp-11h]
  unsigned int v5; // [esp+1Ch] [ebp-10h]

  v5 = __readgsdword(0x14u);
  if ( string_length(a1) != 6 )
    explode_bomb();
  v1 = 0;
  do
  {
    v3[v1] = array_3141[*(_BYTE *)(a1 + v1) & 0xF];
    ++v1;
  }
  while ( v1 != 6 );
  v4 = 0;
  if ( strings_not_equal(v3, "flames") )
    explode_bomb();
  return __readgsdword(0x14u) ^ v5;
}

首先解读下题目，当我们键入字符之后，先进行判断字符数是否为6.
然后是将空数组v3进行连续赋值，最后赋值之后的字符串要与给定的字符串相等。
重点就是这个赋值过程了。可以明显的看到，这个赋值过程是将array_3141数组中的值按照某个规则取出赋值。
在IDA中可以直接查看目标的地址和内容。源地址查看的是16进制数字。转化成ASC码如下

m a d u i e r s n f o t v b y l

如果要使连续赋值等于目标字符串 flames ，我们就需要令6次赋值中array_3141数组角标依次等于

9 15 1 0 5 7

最后的关键就是运算规则了

*(_BYTE *)(a1 + v1) & 0xF

通过查找资料发现 (_BYTE *)

的意义是将目标转化成byte类型的指针，然后前面的那个 * 又取得该地址的数据。
(a1 + v1) & 0xF的含义是，每次将a1数组的脚标为v1的数据进行一次与运算。当然这个运算的具体结构我目前并不清楚。
在c语言中复现代码

int main()
{
    char a = 'o';
    printf("%x\n", a);
    printf("%x\n", a&(0Xf));
        return 0;
    
}

通过多次输入并比较，我们可以粗浅地发现一个结论，这个运算大概是会将字符的16进制代码减少60并处以16取余数
比如检验字符 a 时，运算的结果是 1 ，输入o运算结果是 f （15），输入p运算结果为0.
然后接下来的事情就很简单了，要令其脚标满足条件，只需要在开始输入我们的按照反推运算规则确定的数即可。
