本系列是博主在OUC进修前做新生测试培训的一些心得、思路和学习体会。希望能在新的人生阶段更加努力地进步，获得自己的成长与成就。

题目

题目是一个linux可执行文件，一共六个步骤需要特定的输入，否则将会爆炸。

题目分析

将其拖入IDA32位反汇编，查看主函数伪代码如下。

int __cdecl main(int argc, const char **argv, const char **envp)
{
  const char *v3; // eax
  const char *v4; // eax
  const char *v5; // eax
  const char *v6; // eax
  const char *v7; // eax
  const char *v8; // eax

  if ( argc == 1 )
  {
    infile = (FILE *)stdin;
  }
  else
  {
    if ( argc != 2 )
    {
      __printf_chk(1, "Usage: %s [<input_file>]\n", *argv);
      exit(8);
    }
    infile = fopen(argv[1], "r");
    if ( !infile )
    {
      __printf_chk(1, "%s: Error: Couldn't open %s\n", *argv, argv[1]);
      exit(8);
    }
  }
  initialize_bomb();
  puts("Welcome to my fiendish little bomb. You have 6 phases with");
  puts("which to blow yourself up. Have a nice day!");
  v3 = read_line();
  phase_1((int)v3);
  phase_defused();
  puts("Phase 1 defused. How about the next one?");
  v4 = read_line();
  phase_2((int)v4);
  phase_defused();
  puts("That's number 2.  Keep going!");
  v5 = read_line();
  phase_3(v5);
  phase_defused();
  puts("Halfway there!");
  v6 = read_line();
  phase_4(v6);
  phase_defused();
  puts("So you got that one.  Try this one.");
  v7 = read_line();
  phase_5(v7);
  phase_defused();
  puts("Good work!  On to the next...");
  v8 = read_line();
  phase_6(v8);
  phase_defused();
  return 0;
}

这部分和题目给出的主函数c语言文件差不多，在这里可以直观地看到这道题有6个阶段，如果需要解题的话，结合函数伪代码以及汇编语言分析代码即可。

phase_1

解题过程

查看阶段一伪代码

int __cdecl phase_1(int a1)
{
  int result; // eax

  result = strings_not_equal(a1, "You can Russia from land here in Alaska.");
  if ( result )
    explode_bomb();
  return result;
}

很容易分析出来，这个函数的目的是比较判断。如果输入字符串与预设一致，则不会爆炸。
键入”You can Russia from land here in Alaska.”

phase_2

解题过程

查看阶段二伪代码

int __cdecl phase_2(int a1)
{
  int result; // eax
  char *v2; // ebx
  int v3; // [esp+18h] [ebp-24h]
  char v4; // [esp+1Ch] [ebp-20h]
  char v5; // [esp+30h] [ebp-Ch]

  read_six_numbers(a1, (int)&v3);
  if ( v3 != 1 )
    explode_bomb();
  v2 = &v4;
  do
  {
    result = 2 * *((_DWORD *)v2 - 1);
    if ( *(_DWORD *)v2 != result )
      explode_bomb();
    v2 += 4;
  }
  while ( v2 != &v5 );
  return result;
}

通过查看参数地址，
v3地址为-00000024
v4地址为-00000020
v5地址为-0000000C
查看read_six_numbers函数伪代码

int __cdecl read_six_numbers(int a1, int a2)
{
  int result; // eax

  result = __isoc99_sscanf(a1, (const char *)&unk_804A2FB, a2, a2 + 4, a2 + 8, a2 + 12, a2 + 16, a2 + 20);
  if ( result <= 5 )
    explode_bomb();
  return result;
}

结合之前二阶段主函数的伪代码发现，read_six_numbers目的是将输入的数字存入以v3地址起的连续6个字节内，并判断数字数是否小于等于5，是则爆炸。
循环外有两个步骤，一是判断v3是否为1，我们可以得到的信息是键入的第一个数字为1。二是将v4的地址赋予v2。
循环中需要执行三条语句，分析得。
result的计算值为将v2地址向后移动一个单位的数据翻倍。
v2值等于result。
v2地址加4（一个字节）。
以v2初始值为例，v2需要等于v2的上一个单位的数值即v3的翻倍，这时v2的值等同于v4的值，即得到v4的值。之后v2地址加4，如此循环。
结果是v2的值每次都是上一个单位的值两倍。
故而正确输入是
1 2 4 8 16 32

学习心得

没有使用推荐的工具，主要是我汇编能力不是很好，目前还处在一边百度一边学习的层次。而且IDA用着也比较顺手，但是在我能力提升之后我还是会好好使用别的工具的，比如二阶段那个while循环与我的判断产生了矛盾，但是我并不知道矛盾从何而来。简单的工具一定是有局限性的，这也激励着我们不断像知识的深处摸索。
