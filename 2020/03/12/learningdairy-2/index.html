解题部分

RapidTyping

题目


即，在2.5秒内提交验证码。

解题思路

根据提示“不需要图形学知识”判断，验证码信息应该就储存在网页源码中。
提交任意字符，url出现变化，即通过get方式提交参数。
即，题目与上一题类似，访问源码破译内容并构建get请求提交参数即可。

解题过程

利用开发者工具查看元素得知，图片信息为

<img src="data:image/svg+xml;base64,PHN2ZyB4bWxucz0iaHR0cDovL3.......Zz4=">

base64后跟随着很长一段加密字符，摘取并解密得到可读的图片信息。

<svg xmlns="http://www.w3.org/2000/svg" height="50" width="460"><text x="18" y="37" style="fill: rgb(73, 4, 36);">N</text>....

观察发现，代码信息的主体主要是两个标签。

包含一个字符的

<text x="18" y="37" style="fill: rgb(73, 4, 36);">N</text>

以及

<line x1="301" y1="2" x2="193" y2="8" style="stroke: rgb(105, 130, 34); stroke-width: 0.3px" />

观察验证码的图片发现，图片仅仅包含两个要素，一是没有规律排列的字符，一是各种线条。
此时易得
标签包含线条的数量与位置信息等
标签包含的字符即为验证码本体
通过正则表达式筛选字符并打印

pattern1 = re.compile(r'(?<=;">).*?(?=</text>)')

打印字符与图片不一致，比对后发现顺序不对

通过正则表达式提取坐标x轴信息

pattern2 = re.compile(r'(?<=<text x=")\d{1,3}(?=" y=)')

将两个列表合并成二维列表并按照x轴从小到大排序。
得到验证码，拿到flag。

解题代码

import requests
import re
import base64

url ="http://***********"

s = requests.session()
r = s.get(url)
doc = r.text

pattern = re.compile(r'(?<=base64,).*?(?=" /></p>)')
result = re.findall(pattern,doc)

temp = base64.b64decode(result[0]).decode()

pattern1 = re.compile(r'(?<=;">).*?(?=</text>)')
pattern2 = re.compile(r'(?<=<text x=")\d{1,3}(?=" y=)')

char = re.findall(pattern1,temp)
site = re.findall(pattern2,temp)

site = [ int(x) for x in site ]

zipped = zip(char,site)

res=sorted(zipped,key=(lambda x:x[1]))

list1 = [x[0] for x in res]

ans = ''.join(list1)

学习心得

虽然是web中分最低的几个题之一，但是这个题也花了我不少时间，特别是在最后一步位置排序的时候，因为没有转化成数字，导致顺序错误，提交失败。
为此我白白浪费了很多时间。在之后的训练中，一定要经常会想起做这道题的教训。
如果不能因为熟练而顺利，那就至少要因为细致而强大。
